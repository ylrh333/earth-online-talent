# 地球 Online

一个本地运行的 AI Career Mod 模拟器。

玩家下载仓库后，配置自己的 AI API Key，选择 Career Mod，就可以通过开放式行动体验具体岗位。创作者可以准备某个职业 30 天的真实流水账工作日志，用自己的提示词把日志蒸馏成网页互动游戏 Mod，提交到仓库，后来的人更新仓库后就能玩到更多职业。

这个项目的长期目标是：靠真实从业者的群众力量，把地球上各种具体岗位做成可体验、可比较、可复盘的职业副本，帮助玩家发现自己在哪些真实任务里更自然、更敏锐、更有天赋。

## 一句话

不是职业百科，不是性格测试，而是由真实从业者共创的 AI 职业模拟游戏。

## 概念

- `Career Mod`：职业扩展包，定义一个具体岗位的世界、NPC、任务、事件、评分和 AI 裁判规则。
- `Scenario / Mission`：Career Mod 里的职业副本或任务。
- `Run / Save`：玩家一次游玩的过程和存档。

Career Mod 的真实性来自作者提供的真实岗位材料。作者提供的信息越细，游戏越像真实工作。

生成 Career Mod 前，作者最重要的是准备一份脱敏后的 30 天真实流水账工作日志。格式可以很朴素，像这样：

```text
2026 年 1 月 1 日
上午
1）我打开电脑做了一个全公司职工表格，花费 1 小时。
2）我跟同事一起摸鱼，摸了 30 分钟。
3）领导安排我去现场处理质量问题，花费 1 小时。

下午
1）我到现场确认问题，发现信息不完整。
2）我找研发同事确认标准，但对方说这个问题不归他。

遗留问题
这个质量问题当天没有闭环，第二天还要继续跟。
```

日志不需要写得像报告，越像真实工作流水账越好。建议尽量包含：

- 30 天职业流水账：Day 1 到 Day 30 分别发生什么。
- 每天早上、下午、加班/突发分别做了什么。
- 每一天分别和哪些人打交道，他们各自有什么压力。
- 哪些任务重复出现，哪些问题跨天还没闭环。
- 小白刚进入这个岗位第一天会遇到什么事、什么人。
- 新人最容易在哪里翻车。
- 老手会先看什么、先问什么、不会乱承诺什么。
- 什么表现说明玩家可能有这个岗位的天赋点。
- 脱敏后的真实案例和经历。

## 当前版本

- 本地 Web 应用
- 用户自带 API Key
- OpenAI-compatible Chat Completions 接口
- 无外部 npm 依赖
- 内置官方示例 Mod：汽车质量工程师


打开：

## 生成职业 Mod

主页只保留四个入口：加载职业、生成职业 Mod、文字 API 设置、图片 API 设置。

整个流程很简单：

1. 准备某个职业 30 天的真实流水账工作日志。
2. 自定义“把日志蒸馏成网页游戏 Mod”的提示词。
3. 粘贴日志或上传文档，点击蒸馏按钮生成本地 Career Mod 文件夹。
4. 加载这个 Career Mod，开始玩网页互动游戏。
5. 玩的过程如果感觉不真实，就用右侧 AI Mod 助手实时修改。

点击「生成职业 Mod」后，会打开生成界面。界面上方有一个可编辑的「蒸馏成职业 Mod 的提示词」，默认方法是：先把 30 天流水账里的真实任务、人物、问题、压力和未闭环事项提炼出来，再生成一个更形象、更好玩的真人模拟网页互动游戏 Mod。创作者可以把这段提示词改成自己的蒸馏方法，并保存到本机浏览器。

你可以把 30 天日志直接粘贴到文本框，也可以上传文本或 PDF 文档。系统会把文本框内容和上传文档提取出的文字一起作为素材，生成职业网页游戏 Mod。推荐每一天按这个格式写：

```text
Day 1：
- 早上：谁来找我？给了什么任务、材料、工具或标准？
- 下午：我做了什么？遇到什么信息缺口、判断争议或沟通拉扯？
- 加班/突发：有没有临时催促、返工、会议或现场问题？
- 遇到的人：他们各自想要什么、怕什么、会怎么给压力？
- 判断和行动：我看了什么线索，找了谁，怎么推进？
- 结果或尾巴：当天解决了什么，留下什么未闭环问题？
```

不需要一次写满 30 天。可以先写 Day 1-3，再分段补 Day 4-7、Day 8-15、Day 16-23、Day 24-30。日志和文档越具体，生成的职业 Mod 越真实。信息够了以后，点击「蒸馏生成游戏 Mod」，新职业会保存到本机 `career-mods/<mod-id>/` 文件夹，并出现在「加载职业」下拉框里。

生成出来的网页游戏不应该只是文字问答。一个好的 Career Mod 应该像“真人模拟网页游戏”：有清晰的工作现场图、有具体 NPC、有正在发生的问题、有时间压力、有玩家输入行动后的后果推进。

`career-mods/<mod-id>/` 是完整可分享单元。里面会包含：

- `mod.json`：职业 Mod 元信息。
- `world.md`、`player.md`、`knowledge.md`、`system-prompt.md`、`endings.md`：网页游戏世界、玩家身份、知识库和主持规则。
- `roles.json`、`missions.json`、`events.json`、`scenes.json`、`scoring.json`：NPC、任务、事件、场景和评分。
- `assets/`：生成出来的本地场景图片资产。

别人只要下载这个 Mod 文件夹并放进 `career-mods/`，就能加载基础网页互动游戏内容；如果不接 API，可以玩本地演示模式，如果接入自己的 API，就能获得更强的 NPC 和 AI 裁判。

## 边玩边修 Mod

职业 Mod 的真实性不是一次生成就完成的，而是靠真实从业者试玩、反馈、修订。

点击「加载职业」后，会弹出职业网页游戏界面。游戏界面右侧有「AI Mod 助手」。试玩时如果发现场景、人物、问题、流程或图片提示不真实，可以直接写反馈，例如：

```text
这个场景不真实。汽车质量测试第一天应该先跟着老员工拿标准表检查间隙段差、发动机噪音、音响、底盘、动力、外观、内饰，然后填写问题表并联系研发。
```

点击「根据反馈修改 Mod」后，AI 会修改当前 Career Mod 的任务、NPC、事件、场景画面提示、知识库和评分标准，并保存成 `career-mods/` 里的新 Mod 文件夹。继续试玩、继续反馈，Mod 会越来越贴近真实岗位。

没有 API Key 时也可以用本地演示修订流程，但那不是真实 AI 生成。

## API 设置

在页面上方填写：

- `文字 API 设置`：用于生成职业 Mod、试玩 NPC 对话、根据反馈修改 Mod、分析 Run/Save。
- `图片 API 设置`：用于生成职业网页游戏的场景图。Cockpit Tools 可填 `http://127.0.0.1:60587/v1` 和 `gpt-image-2-medium`，也可以使用其他 OpenAI-compatible Images 服务。

Key 只保存在本机浏览器 localStorage，不写入仓库。请求会经过本地 Node 服务代理，避免浏览器跨域问题。

## 为什么不由项目提供 API Key

因为这是开源项目，不是中心化付费平台。项目开发者不提供公共 API，也不承担玩家的 AI 调用成本。

玩家自己选择模型和服务商，自己输入 API Key。这样仓库可以专注于职业 Mod 框架、玩法和内容质量。

## 创作原则

- 写具体岗位，不写泛行业。
- 写真实压力，不写职业百科。
- 写 NPC 冲突，不让所有人都配合玩家。
- 写玩家行动的后果，不只给知识点。
- 不提交真实公司、客户、供应商、同事、报价、图纸、内部文件或商业机密。

## 贡献职业 Mod

贡献流程见：

```text
CONTRIBUTING.md
```

最推荐的方式是：

1. 先用 `docs/职业Mod规范说明.md` 里的提示词让 AI 生成初稿。
2. 再由真实从业者手动修改，补充行业细节、压力、灰度和边界。
3. 本地试玩和校验。
4. 提交 Pull Request。
